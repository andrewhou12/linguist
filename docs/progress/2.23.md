## Overview

19 commits since the last progress update (`ae629af`). ### 6. Monorepo Migration & Web App (`475d962`, `54acc9c`, `1cf3d30`)

The single largest change — restructured the entire project from a flat Electron app into a Turborepo monorepo with a new Next.js web app.

#### Monorepo Structure

**`pnpm-workspace.yaml`** — Defines workspace packages:
```
packages:
  - "apps/*"
  - "packages/*"
```

**`turbo.json`** — Build orchestration with dependency-aware `build`, persistent `dev`, and `typecheck` tasks.

**Root `package.json`** — Converted to workspace root with shared scripts (`dev:desktop`, `dev:web`, `build`, `typecheck`, `db:migrate`, `db:generate`).

#### File Moves (No Code Changes)

All existing code relocated into the monorepo structure:

| From | To |
|---|---|
| `electron/` | `apps/desktop/electron/` |
| `src/` | `apps/desktop/src/` |
| `core/` | `packages/core/src/` |
| `shared/` | `packages/shared/src/` |
| `core/logger.ts` | `packages/core/src/logger.ts` |
| `electron/logger.ts` | `apps/desktop/electron/logger.ts` |

Each package got its own `package.json` and `tsconfig.json`:
- `packages/core/` → `@lingle/core` (pure TypeScript business logic)
- `packages/shared/` → `@lingle/shared` (TypeScript types only)
- `packages/db/` → `@lingle/db` (Prisma client singleton)
- `apps/desktop/` → `@lingle/desktop` (Electron app)
- `apps/web/` → `@lingle/web` (Next.js app)

Logger imports updated from relative paths to `@lingle/core` package imports across `packages/core/src/`.

## Current Project State

### What's Working End-to-End
- **Dashboard**: Due count, weekly stats, AI daily brief, 3 frontier visualizations (radar, level bars, dot map)
- **Review**: Full SRS with recognition/production cards, keyboard shortcuts, FSRS scheduling, mastery state machine, session summary
- **Learn**: Curriculum preview → session planning (Claude API) → conversation → post-session analysis → results summary
- **Knowledge Base**: Searchable/filterable vocabulary table with mastery badges, due dates, stability
- **Chat**: Streaming conversation with Claude
- **Settings**: Language selection, daily limits, target retention, progress stats
- **Onboarding**: 4-step placement wizard (language → level → adaptive test → preferences), seeds 50–200 items
- **Auth**: Google OAuth via Supabase, session management, protected routes
- **Web App**: Full feature parity with desktop — all pages and API routes functional
- **Logging**: Structured logging across 24 backend files

### Known Issues / Gaps
- Learn page shows item IDs instead of surface forms in session summary (carried over from 2/19)
- Word bank detail view (editing, review history) not built
- No error boundaries or toast notifications — errors go to console only
- Session planning latency (Claude API) has no progress indicator beyond loading state
- The `_sessionId` type hack on session plan persists
- Chat page is disconnected from the knowledge model (no profile awareness, no post-session analysis)
- Insights page is a placeholder on both platforms
- Desktop and web share no code at the React component level — components were duplicated rather than shared

### Architecture Health
- **`core/` boundary intact**: Zero Electron/Prisma/React imports across all 21 core files. Package extraction validated by monorepo structure.
- **IPC/API parity**: Every IPC handler has a corresponding HTTP route with identical logic.
- **Multi-tenancy complete**: Every DB query scoped by `userId`. No data leakage paths.
- **Build system working**: Turborepo orchestrates builds across all 5 packages with dependency ordering.
- **Auth integrated**: Both platforms use Supabase Auth — Electron via direct client, web via SSR cookies.

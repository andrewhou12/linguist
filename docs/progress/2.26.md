# Session 7: MVP v0 — Conversation, Knowledge Base, Session History (2026-02-26)

## Overview

The codebase shifted from "feature parity across platforms" to **shipping a usable MVP**. This sprint implemented the 3 primary surfaces defined in the [MVP spec](../plans/mvp-spec.md): **Conversation** (with structured cards, streaming, and a learning loop), **Knowledge Base** (3-tab vocabulary/grammar/phrases view), and **Session History** (annotated transcripts). The sidebar was trimmed to these 3 surfaces + Settings, and `/conversation` became the default landing page.

The core architectural win: the conversation pipeline now closes the loop — FSRS scheduling and mastery transitions happen automatically at session end, so every conversation produces durable learning state updates.

## What Was Built

### 1. Learning Loop Completion (Phase 1)

**FSRS + mastery transitions in conversation/end** — `apps/web/app/api/conversation/end/route.ts`

Previously, the post-session analysis logged context but didn't update FSRS or mastery states. Now:
- For each production event (successful or not), `scheduleReview()` is called with the appropriate grade (`good` for success, `again` for failure)
- `computeNextMasteryState()` evaluates whether the item should advance (e.g., apprentice_4 → journeyman with sufficient production evidence)
- A `ReviewEvent` is created for each FSRS update with `contextType: 'conversation'`
- Item existence is validated before creating context logs (guards against hallucinated IDs from the analysis model)

**Streaming conversation responses** — `apps/web/app/api/conversation/send/route.ts`

Rewrote to support SSE streaming:
- When `stream: true` in the request body, uses `anthropic.messages.stream()` and returns a `ReadableStream` with SSE events
- Event types: `{ type: 'delta', text }`, `{ type: 'done', message }`, `{ type: 'error', error }`
- Non-streaming fallback preserved for backwards compatibility
- Full response accumulated server-side and saved to transcript after stream completes

**Structured card instructions in system prompt** — `packages/core/src/conversation/planner.ts`

Added 7 new behavioral rules (11–17) to `buildConversationSystemPrompt()`:
- `[VOCAB_CARD]` blocks for vocabulary introduction (surface, reading, meaning, example)
- `[GRAMMAR_CARD]` blocks for grammar explanation (pattern, meaning, formation, example)
- `[CORRECTION]` blocks for error correction (incorrect → correct + explanation)
- `[REVIEW_PROMPT]` blocks for recall testing
- `[TARGETS_HIT: item1, item2]` metadata line for real-time challenge tracking
- Card budget system: `<20 active items → 5 cards, 20–80 → 3, 80+ → 1`
- Turn 20 wind-down: no new vocabulary or grammar after this point

**Updated conversation partner framing** — removed persona/character instructions, replaced with "You are a Japanese language practice partner" focused on natural conversation at the learner's level.

## Bugs Fixed During Testing

**withAuth masking handler errors as 401s** — The catch-all in `withAuth` was returning 401 for any error, including Prisma errors inside the handler. Split into separate auth/handler error paths so real errors surface with 500 status.

**Missing `chunkItemId` column on ReviewEvent** — Prisma client was regenerated but the database schema hadn't been pushed. Fixed with `npx prisma db push`.

**Foreign key constraint violation on ItemContextLog** — The post-session analysis model (Claude) sometimes returns item IDs that don't exist in the database. Added existence checks before creating `ItemContextLog` entries — if the referenced lexical/grammar item doesn't exist, the log is skipped rather than crashing.

## Current Project State

### What's Working End-to-End
- **Conversation**: Planning → streaming chat with structured cards → real-time challenge tracking → session end with FSRS updates → summary
- **Knowledge Base**: 3-tab view (vocabulary, grammar, phrases) with search, mastery filter, expandable detail, "Start Practicing"
- **Session History**: Session list with challenge completion ratios → click for annotated transcript with matched annotations
- **Learning loop**: Conversation production evidence flows through FSRS scheduling → mastery state machine → durable knowledge state updates
- **Onboarding**: Seeds vocabulary, grammar, AND chunks on completion
- **Navigation**: Clean 3-surface sidebar with Settings

### Known Issues / Gaps
- Card rendering depends on Claude following the `[TAG]...[/TAG]` format consistently — no fallback if it doesn't
- Challenge card shows item IDs rather than surface forms for some items
- No error boundaries or toast notifications — errors still go to console only
- Session summary in history view shows raw analysis data, could be richer
- Streaming interruption (user navigates away mid-conversation) not handled gracefully
- Desktop app has not been updated with these changes (web-only sprint)

### What to Build Next
1. **Testing** — End-to-end conversation loop verification, structured card parsing tests
2. **Error handling UX** — Toast notifications, error boundaries, retry logic
3. **Card rendering reliability** — Fallback behavior when agent doesn't emit structured blocks
4. **Desktop parity** — Port conversation UI, knowledge tabs, history to Electron
5. **Session quality metrics** — Track card budget adherence, target hit rate over time
6. **Voice layer (V2)** — STT → LLM → TTS pipeline per the architecture spec

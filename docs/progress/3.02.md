# 2026-03-02 — UI Overhaul: Conversation Landing, Navigation, API Caching, Analysis Precision

## Summary

Major UI/UX update spanning the conversation landing page redesign, navigation modernization, client-side API caching with stale-while-revalidate, post-session analysis precision improvements, and several quality-of-life fixes across the codebase.

---

## Changes by Area

### 1. Conversation Landing Page Redesign

**Files:** `apps/web/components/conversation-view.tsx` (new), `apps/web/app/(app)/conversation/page.tsx`, `apps/web/app/globals.css`

The conversation page's planning phase was redesigned from a utilitarian "Ready to practice?" screen into a ChatGPT-style landing experience:

- **Extracted `ConversationView` into its own component** (`components/conversation-view.tsx`), rendered persistently in the app layout to preserve state across navigations. The old `conversation/page.tsx` (638 lines) was replaced with a 6-line stub.
- **Stylized "Lingle" logo** with a `logo-shimmer` CSS animation — warm accent gradient sweep via `background-clip: text`, 5-second cycle.
- **Time-of-day Japanese greeting**: おはよう / こんにちは / こんばんは based on local time.
- **Contextual subtext**: Shows the most salient piece of context — avoidance patterns > streak count > weekly review count.
- **Front-and-center ChatInput** with `minRows={3}` for a taller input area. Typing a message and pressing Enter starts a session with that message as the topic and first utterance.
- **Topic starter cards** (2x2 grid): Cafe ordering, Weekend plans, City exploration, Free conversation. Each has a pre-written Japanese first message and a planner hint so the session plan incorporates the chosen topic.

**Planner integration:**
- `packages/core/src/conversation/planner.ts` — `buildPlanningPrompt()` now accepts optional `topicHint?: string`, injected before the JSON schema instruction.
- `apps/web/app/api/conversation/plan/route.ts` — Parses `topicHint` from request body (graceful fallback if missing).
- `apps/web/lib/api.ts` — `conversationPlan(topicHint?)` passes the hint as JSON body.

### 2. Navigation Modernization

**Files:** `apps/web/app/(app)/layout.tsx`, `docs/design-system.md`

- **Replaced emoji nav icons with Lucide icons** (`MessageCircle`, `Zap`, `BookOpen`, `PenLine`, `BookMarked`, `Clock`, `Flame`, `Settings`, `BarChart3`). More consistent, better alignment, smaller visual footprint.
- **Updated nav link styles**: `font-medium` always, tighter padding (`py-1.5 px-3`), `rounded-md` instead of `rounded-lg`.
- **Progress widget** also switched from emoji to Lucide icons.
- Design system doc updated to match.

### 3. Client-Side API Caching (Stale-While-Revalidate)

**Files:** `apps/web/lib/api.ts`, `apps/web/hooks/use-wordbank.ts`, `apps/web/hooks/use-grammar.ts`, `apps/web/hooks/use-chunks.ts`, `apps/web/hooks/use-review.ts`, `apps/web/hooks/use-frontier.ts`, `apps/web/app/(app)/dashboard/page.tsx`

- **`LingleApiClient`** gained a `cache` Map and `inflight` Map for deduplication. GET requests return cached data instantly and revalidate in the background after 30 seconds (`STALE_AFTER`). Mutations (`POST`, `PATCH`) invalidate the entire cache.
- **`peekCache<T>(path)`** — Synchronous cache read used by hooks to initialize React state without a loading flash.
- All data hooks (`useWordbank`, `useGrammar`, `useChunks`, `useReview`, `useFrontier`) now initialize from cache via `peekCache` and skip the initial loading spinner when data is already cached.
- **`prefetch()`** method preloads all common page data on app startup.
- Dashboard page initializes state from cache for instant rendering.

### 4. Post-Session Analysis — Message-Level Precision

**Files:** `packages/core/src/conversation/analyzer.ts`, `packages/shared/src/types.ts`, `apps/web/app/(app)/history/[id]/page.tsx`, `apps/web/components/session-summary.tsx`, `apps/web/components/chat/session-summary-modal.tsx`

- **Numbered transcript**: The analysis prompt now sends `[index] [role] content` per message, and asks Claude to include `message_index` in `targets_hit`, `errors_logged`, and `avoidance_events`.
- **`TargetHitEntry` type** added to shared types — `{ itemId, messageIndex?, contextQuote? }`. `PostSessionAnalysis.targetsHit` changed from `number[]` to `Array<number | TargetHitEntry>` for backwards compatibility.
- **History detail page** now annotates the correct message using `messageIndex` (falling back to substring matching for old sessions). User messages now also show annotations (target hits, errors).
- **Session summary modal** and **session summary card** updated to handle both old `number[]` and new `TargetHitEntry[]` formats.

### 5. Selection X-Ray

**Files:** `apps/web/components/chat/selection-xray-popover.tsx` (new), `apps/web/hooks/use-selection-xray.ts` (new), `apps/web/components/chat/sentence-xray.tsx`

- **Selection X-Ray popover**: Select any Japanese text in the conversation → popover appears with token-level breakdown (surface, reading, romaji, POS, meaning).
- Extracted `XRayTokenGrid` and `posColors` from `sentence-xray.tsx` into reusable exports.
- Added tooltip hint on the X-Ray button: "X-Ray (or select text + Cmd+E)".

### 6. Streaming Cursor Fix

**Files:** `apps/web/components/chat/message-block.tsx`, `apps/web/components/conversation-view.tsx`

- Fixed a bug where the blinking cursor during streaming appeared above the text (in an empty `content` div) instead of inline with the streamed text.
- `MessageBlock` no longer renders a cursor-only div when `content` is empty.
- Cursor now renders inline at the end of the last text `PartRenderer` during streaming, using `[&>p:last-of-type]:inline` CSS to flow after Markdown's block-level output.

### 7. ChatInput `minRows` Prop

**File:** `apps/web/components/chat/chat-input.tsx`

- Added `minRows?: number` prop (default `1`), replacing the hardcoded `rows={1}`. Used by the conversation landing page with `minRows={3}`.

### 8. Miscellaneous Fixes

- **X-Ray API** (`apps/web/app/api/conversation/xray/route.ts`): Increased `maxOutputTokens` from 1024 to 4096. Added try/catch around JSON parse with 502 error response on failure.
- **Conversation end route** (`apps/web/app/api/conversation/end/route.ts`): Fixed Prisma type cast for `targetsHit`.
- **IME default state** (`apps/web/hooks/use-japanese-ime.ts`): IME now defaults to active on first use (this is a Japanese learning app).
- **Kana dictionary** (`apps/web/lib/kana-dictionary.ts`): Expanded hiragana-preferred word list (adverbs, conjunctions, te-forms). Added te-form boundary splitting and particle boundary checking to prevent greedy matches like `してい→指定` instead of `して+います`.
- **Page-level cleanup**: Dashboard, history, review, settings, and learn pages received minor fixes (cache initialization, border color tokens, type casts).
